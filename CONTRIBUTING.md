# Contributing

Open an issue before changing the forecasting protocol or public data contract. Pull requests
must keep year-based leakage barriers intact, add tests for behavior changes, pass Python and
MATLAB CI, and avoid committing private data or generated full-run artifacts.

Claims about model superiority require an artifact manifest, the input SHA-256, shared evaluation
dates, and metrics in original sales units. Fuzzy rules must be described as associations unless
a separate causal design supports stronger language.
